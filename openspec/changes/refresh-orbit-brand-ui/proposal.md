# refresh-orbit-brand-ui

## Why

- 主窗口没有唯一主工作区。`打开 CLI`、账号详情、额度信息在首屏同时抢权重，选中账号后仍像文档页，不像操作台。
- 详情区依赖多块同级卡片。浅色圆角容器被重复使用，形成“组件马赛克”，重要状态和辅助信息没有稳定主次。
- 当前样式更多依赖 `regularMaterial`、浅底色和圆角块来建立分组，Orbit 品牌识别弱，产品名没有成为第一印象。
- 新增账号窗口本质上仍是状态表单。模式切换、解释文案、字段和操作区平铺在一列里，缺少单任务引导感。

## What Changes

- 新增 `refresh-orbit-brand-ui` OpenSpec 变更，使用“Orbit 控制台”作为统一视觉命题，吸收 `frontend-skill` 的克制布局、单一强调色、少卡片和明确层级原则。
- 重新定义主窗口信息架构：左侧账号轨道，中央主工作区，右侧次级检视区；首屏只保留一个绝对主操作，即打开 CLI。
- 重写新增账号窗口的布局规范，使其成为单任务引导面板：顶部模式选择，中部当前流程字段，底部固定主操作。
- 增加最小设计系统约束，包括颜色角色、字体角色、间距、圆角和仅保留三类动效。
- 将 UI 验收条件补充到 `platform-account-management` 变更 spec，确保 `960x620` 下主要操作仍然清晰可达。

## Non-Goals

- 不修改 `AppViewModel`、数据模型、Provider 规则、CLI 启动逻辑或平台能力边界。
- 不单独重构菜单栏面板的信息架构，只要求其样式与新主窗口保持一致。
- 不引入新的组件库、主题引擎或与本次 UI 重塑无关的抽象。
